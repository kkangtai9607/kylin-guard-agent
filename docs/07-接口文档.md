# 接口文档（Phase 0 草案）

## 1. 通用约定

- 前缀：`/api/v1`
- 传输：HTTPS；任务流使用 SSE，后续按必要性评估 WebSocket。
- 认证：服务端安全会话；状态变更接口同时执行 RBAC、同源/CSRF 校验和审计。
- 幂等：审批、取消、基线和设置修改等写接口支持 `Idempotency-Key`。
- 时间：UTC ISO 8601；标识符使用 UUID。
- 分页：`cursor` 与 `limit`，服务端限制最大值。

统一响应：

```json
{
  "request_id": "uuid",
  "data": {},
  "error": null,
  "meta": {"mode": "READ_ONLY", "is_demo": false}
}
```

错误对象只包含稳定 `code`、可公开 `message` 和经过脱敏的 `details`，不返回堆栈、环境变量、Secret 或模型隐藏思维。

## 2. 主要资源

### 身份与系统

- `POST /auth/login`
- `POST /auth/logout`
- `GET /auth/me`
- `GET /system/overview`
- `GET /system/capabilities`

### 任务

- `POST /tasks`
- `GET /tasks`
- `GET /tasks/{task_id}`
- `POST /tasks/{task_id}/cancel`
- `GET /tasks/{task_id}/events`
- `GET /tasks/{task_id}/stream`

任务创建仅允许：

```json
{
  "goal": "分析磁盘空间不足原因",
  "target_scope": {"host_ids": ["local"]},
  "requested_mode": "READ_ONLY"
}
```

不得接受 Shell、命令字符串、任意可执行路径、风险等级、审批结果或“跳过安全检查”等字段。`requested_mode` 只能等于或严于服务端当前模式。

### 审批

- `GET /approvals`
- `GET /approvals/{approval_id}`
- `POST /approvals/{approval_id}/approve`
- `POST /approvals/{approval_id}/reject`

批准请求需要重认证证明和展示给审批者的参数哈希；服务端签发短期、单次、绑定用户/任务/工具/参数的批准令牌。L4 返回永久拒绝。

### MCP、巡检与事件

- `GET /mcp/tools`
- `POST /mcp/tools/{tool_name}/test`：仅运行注册 Tool 的安全自检或 dry-run。
- `POST /inspections/run`
- `GET /inspections`
- `GET /incidents`
- `GET /incidents/{incident_id}`

### 配置漂移与知识库

- `GET /config-drift`
- `POST /config-drift/baselines`
- `GET /knowledge`
- `POST /knowledge`
- `GET /knowledge/{document_id}`
- `PUT /knowledge/{document_id}`

知识内容作为不可信数据保存，并记录来源、审核状态和允许支持的最高风险等级。

### 审计与设置

- `GET /audit/events`
- `POST /audit/verify`
- `GET /audit/export`
- `GET /settings`
- `PUT /settings`

设置接口不能读取或返回 Secret 值，只能显示 Secret 引用是否已配置。

## 3. SSE 事件

SSE 支持 `Last-Event-ID` 断线续传。事件包含 `event_id`、`task_id`、`event_type`、`state`、`timestamp`、公开理由摘要和脱敏 payload；不包含隐藏思维、审批签名或原始凭据。客户端必须处理重复事件。

## 4. 错误码

至少包括：`VALIDATION_ERROR`、`AUTH_REQUIRED`、`RBAC_DENIED`、`MODE_FORBIDDEN`、`RISK_REJECTED`、`TOOL_NOT_REGISTERED`、`CAPABILITY_UNAVAILABLE`、`PATH_REJECTED`、`APPROVAL_REQUIRED`、`APPROVAL_EXPIRED`、`APPROVAL_REPLAYED`、`BACKUP_FAILED`、`EXECUTION_FAILED`、`VERIFICATION_FAILED`、`ROLLBACK_FAILED`、`TIMEOUT`、`OUTPUT_TRUNCATED` 和 `INTERNAL_ERROR`。
