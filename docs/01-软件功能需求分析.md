# 软件功能需求分析

## 1. 产品目标与范围

麒麟智维盾在自然语言与麒麟操作系统之间提供安全桥梁，覆盖系统感知、巡检诊断、根因分析、受控修复、审批和审计。Phase 0 只确定设计，不实现运行能力。

## 2. 用户与角色

| 角色 | 主要能力 | 禁止能力 |
|---|---|---|
| `viewer` | 查看获授权的概览、任务和审计摘要 | 创建任务、审批、执行 |
| `operator` | 创建诊断/巡检任务、查看证据、提出受控操作 | 审批自己的 L3 操作、修改策略 |
| `approver` | 审阅并批准或拒绝 L3 操作 | 绕过规则引擎、批准 L4 |
| `admin` | 用户、角色、非秘密设置和策略管理 | 绕过安全规则、读取凭据 |
| `auditor` | 查询和验证脱敏审计链 | 发起执行或修改审计记录 |

审批权限由后端 RBAC 强制执行，不能只依赖页面隐藏按钮。默认禁止发起人与审批人为同一用户；演示模式若采用单人演示，必须明确显示“演示审批”，并仍执行完整令牌校验。

## 3. 核心用例

1. 查询系统资源、进程、端口和服务状态。
2. 运行安全巡检、配置漂移和故障诊断。
3. 汇总跨数据源证据，输出可解释的 Top 3 根因候选。
4. 对白名单动作生成 dry-run 影响摘要并提交人工审批。
5. 在最小权限下执行、验证，失败时回滚。
6. 查询任务时间线、审批和防篡改审计事件。
7. 拒绝凭据读取、任意命令、破坏性或越权请求。
8. 隔离日志和知识库中的间接提示词注入。

## 4. 功能需求

- FR-01：API 必须完成认证、RBAC、请求追踪和统一错误处理。
- FR-02：Agent 必须使用显式状态机和严格 `ActionPlan` Schema。
- FR-03：系统必须支持 `DEMO`、`READ_ONLY`、`CONTROLLED_EXECUTION` 三种模式，默认 `READ_ONLY`。
- FR-04：系统感知必须通过已注册的固定 MCP Tool 完成，不提供通用 Shell Tool。
- FR-05：风险等级由确定性规则引擎给出最终结果，LLM 不能降低风险。
- FR-06：所有 L3 操作必须人工审批；L4 不得审批放行。
- FR-07：受控写操作必须具备 dry-run、备份、验证、失败回滚和完整审计。
- FR-08：所有外部文本进入模型前必须被标记为 `UNTRUSTED_DATA`、脱敏、截断并保留来源。
- FR-09：只保存结构化决策链，不保存模型隐藏思维过程。
- FR-10：知识库默认使用 SQLite FTS5；向量检索只能作为可选增强。
- FR-11：审计事件必须可检索、导出、脱敏并支持哈希链完整性验证。
- FR-12：DEMO 数据必须在接口、页面、导出和审计中明确标识。

## 5. 非功能需求

- 安全：最小权限、默认拒绝、安全失败、Secret 零落盘和纵深防御。
- 确定性：状态迁移、风险、审批和执行资格均由服务端规则决定。
- 兼容：Python 3.10+；不将 GPU、本地模型、向量数据库、Docker 或 x86 专属二进制作为硬依赖。
- 可用：关键能力探测失败时明确降级，不伪造真实系统结果。
- 性能：Tool 有超时、输出长度和并发限制；SQLite 使用 WAL 和短事务。
- 可演示：固定种子和可恢复场景应在 6 分 30 秒内完成主流程。
- 可维护：需求、API、ER、Tool 元数据和评分映射必须保持可追踪。

## 6. 验收边界

Phase 0 验收架构和文档一致性，不验收真实 API、MCP、LLM、数据库或 UI。麒麟 V11 和 LoongArch 仅形成兼容矩阵，真机结论留到 Phase 9。
