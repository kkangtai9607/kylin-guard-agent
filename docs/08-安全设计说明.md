# 安全设计说明

## 1. 安全目标

系统在模型误判、提示词注入、恶意输入、污染知识和组件故障时仍应默认拒绝危险动作。模型负责理解、规划、选择注册 Tool 和生成公开摘要，但没有最终执行权限。确定性规则引擎独立于主模型和审计模型，并拥有最终否决权。

## 2. 风险等级

| 等级 | 定义 | 处置 |
|---|---|---|
| L0 | 无系统访问的知识问答 | 直接响应并记录摘要 |
| L1 | 基础只读查询 | RBAC 和范围校验后允许 |
| L2 | 组合诊断或敏感元数据读取 | 限制范围、脱敏、限流、完整审计 |
| L3 | 白名单受控状态变更 | 仅 CONTROLLED_EXECUTION；人工审批、备份、验证、失败回滚 |
| L4 | 凭据、破坏性、越权、逃逸或任意命令 | 永久拒绝，不可通过审批降级 |

最终风险为工具固有风险、参数提升风险、目标资源风险、用户意图风险和模式约束中的最高值。LLM 标签只能提高关注度，不能降低规则结果。

## 3. 双层护栏

第一层是确定性规则：Schema、Tool 白名单、模式、RBAC、路径、参数、资源、审批、备份、验证和回滚约束。第二层是可选审计模型，用于发现意图偏移和过度执行。第二层不可放行第一层拒绝的动作；第二层失败时安全降级。

Guardrail 只向 Executor 输出内部 `AuthorizedExecution`：绑定任务、Tool、规范化参数哈希、风险、审批令牌引用和有效期。Executor 拒绝 ActionPlan、自然语言、LLM 原始 JSON、命令字符串或任意可执行路径。

## 4. 任意 Shell 与参数安全

- 禁止 `shell=True`、字符串拼接执行、`eval`、`exec` 和通用 Shell Tool。
- Tool 内只允许固定绝对可执行路径或受控 PATH，使用 argv 列表和 `shell=False`。
- Pydantic/JSON Schema 严格拒绝多余字段，限制枚举、长度、范围和格式。
- 路径先规范化，再检查 allowed roots、protected paths 和符号链接；审批后执行前重新验证目标。
- PID、服务、端口和文件对象均执行资源级保护，禁止终止自身、PID 1 和关键系统进程。

## 5. 人工审批与执行生命周期

所有 L3 写操作必须人工审批。令牌使用 HMAC 签名、短 TTL、单次 nonce，并绑定 `user_id`、`task_id`、`tool_name`、规范化参数哈希和过期时间。参数、工具或目标变化立即失效；令牌原子消费，防止重放。

执行固定顺序：

```text
ACTION_GUARD → DRY_RUN → WAITING_APPROVAL → APPROVED
→ BACKING_UP → EXECUTING → VERIFYING → SUCCEEDED
```

备份失败不得执行；验证失败进入回滚，回滚后再次验证。回滚失败生成高严重度事件并停止，不进行未经批准的补救操作。

## 6. 直接与间接提示词注入

用户输入、日志、配置、进程命令行、网络返回、知识文档和历史案例全部是不可信数据。进入模型前必须：

- 放入带来源、内容类型、采集时间、长度和内容哈希的结构化信封。
- 明确标记 `UNTRUSTED_DATA`，不能拼入 system/developer 指令。
- 脱敏、截断、清理控制字符，并保留原始来源引用。
- 将其中命令视为普通文本，不能改变 Tool 策略、风险、模式或审批。
- 对 Tool 选择和参数在模型之后重新进行确定性校验。

直接请求读取 `/etc/shadow`、SSH 私钥、Token 或 API Key 必须在 Input Guard 阶段作为 L4 阻断且不调用系统 Tool。知识库内容按来源和审核状态分级，未审核案例不能支持 L3 动作。

## 7. 凭据与隐私

- Secret 只从环境变量或系统 Secret 注入，不进入 YAML、数据库、日志、前端或审计。
- `.env.example` 仅含空值；已经在聊天或非保密渠道出现的 Key 必须撤销并轮换。
- 字段名匹配 key、token、password、secret、authorization、private key 时默认脱敏。
- Tool 输出和异常使用字段白名单，限制长度并禁止返回敏感文件内容。

## 8. 决策链与审计

不得请求、保存或展示模型隐藏思维过程。可追溯内容限定为用户目标、意图类别、系统证据、检索案例、结构化计划、Tool 调用、风险判定、审批记录、执行、验证、回滚和可公开理由摘要。

审计事件使用 canonical JSON 构造哈希链：

```text
current_hash = SHA256(canonical_json(event_without_hash) + previous_hash)
```

事件只追加，支持时间线、条件检索、脱敏导出和完整性校验。

## 9. 最小权限与兼容

API 使用 `kylin-guard` 账号，Restricted Executor 使用 `kylin-guard-exec`。后端不长期以 root 运行；sudoers 只允许极少固定入口，不允许参数通配或通用解释器。不依赖 GPU、本地模型、向量数据库、Docker 或 x86 专属二进制。

详细威胁枚举见[威胁模型](architecture/威胁模型.md)。
