# 软件功能设计

## 1. 模块职责

| 模块 | 职责 | 安全边界 |
|---|---|---|
| Web UI | 驾驶舱、对话、任务时间线、审批、插件和审计展示 | 不决定风险或执行资格 |
| API/Auth | REST/SSE、认证、RBAC、请求追踪、幂等 | 不接受命令字符串 |
| Agent Orchestrator | 意图、复杂度、结构化计划、证据编排和公开摘要 | 输出是候选数据 |
| LLM Provider | 可替换模型适配、结构化生成、超时和健康检查 | 无 Executor 访问权 |
| Safety Guardrail | Schema、模式、角色、风险、路径、参数和意图一致性校验 | 独立最终否决 |
| MCP Registry/Client | 固定 Tool 元数据与调用路由 | 仅注册 Tool 可调用 |
| Restricted Executor | 固定 argv、最小权限、资源限制和执行生命周期 | 不接收 LLM 原始输出 |
| Evidence/RCA | 证据标准化、规则关联、Top 3 可解释评分 | 置信度不可由 LLM 任意生成 |
| Knowledge | FTS5 文档与已审核案例检索 | 未审核内容不能支持高风险执行 |
| Audit | 脱敏事件、哈希链、验证与导出 | 只追加，Secret 不入库 |

## 2. 数据流

用户目标先经过认证和 Input Guard，再由 Agent 生成严格结构化计划。计划通过确定性 Guardrail 后，只能调用注册 MCP Tool。Tool 输出形成证据，并以不可信数据封装进入 RCA/LLM。任何写操作需再次通过动作 Guard、dry-run 和人工审批，随后由 Restricted Executor 完成备份、执行、验证和回滚。

详细流程见[业务闭环](architecture/业务闭环.md)，风险和边界见[安全设计说明](08-安全设计说明.md)。

## 3. 运行模式

| 能力 | DEMO | READ_ONLY | CONTROLLED_EXECUTION |
|---|---|---|---|
| 数据 | 固定模拟数据 | 真实只读数据 | 真实数据 |
| L1/L2 | 允许 | 允许 | 允许 |
| L3 | 仅隔离演示资源，模拟审批但真实校验 | 拒绝 | 强制审批后允许白名单动作 |
| L4 | 拒绝 | 拒绝 | 拒绝 |
| 标识 | `is_demo=true` | `mode=READ_ONLY` | 高风险模式提示 |

模式来自受信任启动配置，任务和模型均不得提升模式。配置缺失或无效时安全失败，绝不默认开放写权限。

## 4. 降级策略

- LLM 不可用：只读确定性 Tool 仍可运行；涉及规划的任务返回明确降级状态。演示可使用标记为离线夹具的预录结构化响应。
- MCP 不可用：任务进入 `BLOCKED`，不伪造证据。
- 审计模型不可用：确定性规则照常执行，默认不放行可疑动作。
- 平台命令缺失：能力探测后使用 `/proc`、`/sys` 或替代 Tool；否则返回 `CAPABILITY_UNAVAILABLE`。
- 备份失败：禁止执行。
- 验证失败：自动进入回滚；回滚失败产生高严重度事件并停止。
- 数据库繁忙：短事务、WAL、受控 busy timeout；超时后安全失败。

## 5. LLM Provider 设计约束

统一接口规划为 `generate_structured()`、健康检查、超时、有限重试、流式能力和 Schema 能力声明。DeepSeek 为默认 Provider，模型名与 Base URL 配置化，同时预留 OpenAI-compatible 和其他 Provider。厂商 SDK 不得侵入 Guardrail、MCP 或 Executor。
