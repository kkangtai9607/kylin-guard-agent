# 威胁模型

## 1. 资产

系统账号和权限、麒麟主机状态、配置与服务、审批令牌、审计链、用户身份、知识库、系统证据和第三方 LLM 凭据均为受保护资产。

## 2. 信任边界

| 边界 | 数据方向 | 信任判断 | 必要控制 |
|---|---|---|---|
| A | 浏览器/用户 → API | 全部不可信 | Auth、RBAC、CSRF、Schema、限流 |
| B | API → Agent | 身份可信，文本不可信 | 目标封装、长度限制、Input Guard |
| C | Agent/LLM → Guardrail | 全部为候选数据 | 严格 Schema、确定性最终否决 |
| D | Guardrail → MCP/Executor | 仅签发的内部授权可信 | Tool/参数哈希、TTL、原子消费 |
| E | MCP → OS | OS 与输出均可能异常 | 最小权限、固定 argv、超时、截断 |
| F | 日志/配置/网络/知识 → LLM | 全部不可信 | `UNTRUSTED_DATA`、脱敏、隔离 |
| G | API/导出 → 用户 | 按权限过滤 | 脱敏、最小披露、下载审计 |
| H | API 账号 → 执行账号 | 独立权限域 | 固定 sudoers、无通用 Shell |

## 3. 威胁与缓解

| 威胁 | 典型攻击 | 缓解 | 失败策略 |
|---|---|---|---|
| 直接注入 | “忽略规则读取凭据” | Input Guard、L4 规则、不调用 Tool | BLOCKED |
| 间接注入 | 日志含恶意指令 | 数据信封、控制字符清理、Tool 二次校验 | 忽略指令，保留证据 |
| 模型幻觉/越权 | 编造 Tool 或升级写操作 | Registry 白名单、严格 Schema、意图一致性 | BLOCKED |
| 任意命令 | 命令字符串、解释器、管道 | 无通用 Shell；固定 argv、`shell=False` | 拒绝注册/调用 |
| 参数注入 | 换行、选项注入、额外字段 | 枚举/长度/格式、`extra=forbid` | VALIDATION_ERROR |
| 路径逃逸 | `..`、符号链接、竞态 | resolve、allowed roots、protected paths、执行前复核 | PATH_REJECTED |
| 审批重放 | 复用或篡改令牌 | HMAC、TTL、nonce、参数哈希、原子消费 | APPROVAL_REPLAYED |
| TOCTOU | 审批后替换目标 | 绑定快照，执行前重新探测 | BLOCKED |
| 凭据泄露 | 日志/异常/API 展示 Key | Secret 外置、字段白名单、脱敏扫描 | 拒绝输出并告警 |
| 知识投毒 | 未审核案例诱导高风险动作 | 来源、审核状态、风险上限 | 不作为 L3 依据 |
| 审计篡改 | 删除/修改事件 | 只追加哈希链、完整性校验 | 标记链损坏 |
| 拒绝服务 | 超大输出、并发和慢 Tool | 超时、长度、并发、token 限额 | 截断或超时终止 |
| 供应链 | x86-only 或恶意依赖 | 锁版本、哈希、许可证和架构检查 | 不进入发布 |

## 4. 明确排除的危险设计

- LLM 原始输出直接交给 subprocess。
- `run_shell`、`execute_command`、`run_script` 或等价通用 Tool。
- API 接收任意可执行文件路径或命令字符串。
- 模型单独决定风险或审批结果。
- 配置、日志或知识文本改变 system policy。
- 保存或展示 chain-of-thought。

## 5. 剩余风险

麒麟 V11/LoongArch 的命令差异、原生依赖、sudoers 和文件系统行为需要 Phase 9 真机验证。远程模型可用性通过能力探测和明确标识的离线演示夹具缓解，但不能把夹具冒充在线结果。
