# ADR 0002：最小权限固定动作执行代理

日期：2026-07-12  
状态：已采纳，默认关闭

## 决策

生产 API 服务继续以 `kylin-guard` 非 root 账号运行，并保持 `NoNewPrivileges=true`。它不拥有 sudo 权限、不能调用 Shell，也不能直接启动特权命令。

当且仅当管理员完成目标机核验并显式启用受控执行时，API 通过本机 Unix Socket 向 `kylin-guard-exec` 发送严格结构化请求。代理校验 Linux peer UID、请求版本、字段集合、动作和服务名；然后只能通过 sudo 执行 root 所有、不可由服务账号修改的固定辅助程序。

首个最小闭环只允许：

```text
service_restart + nginx -> /usr/bin/systemctl restart nginx
```

root 辅助程序拒绝所有其他参数。sudoers 不含通配符、解释器、Shell 或用户可控参数。

## 后果

- 保持 API 服务隔离；模型输出、HTTP 请求和审批令牌都不能直接变成命令文本。
- 代理不可用时，dry-run 和执行都会安全拒绝，不会自动降级为直接 sudo 或 root API。
- 执行代理通过 POSIX ACL 获得 SQLite 数据目录和审批记录的最小读写权限，以原子消费一次性令牌；不共享 API 的 sudo 权限。目标平台无 ACL 能力时，生产受控执行必须保持关闭。
- `safe_log_cleanup`、`config_safe_update`、`terminate_process` 在真实生产环境仍未开放：它们必须各自具备同等固定的 helper、备份、验证、回滚和目标机演练后才能加入代理白名单。
- DEMO 模式的隔离写操作不受该生产代理替代，仍独立执行完整审批/备份/验证/回滚闭环。
