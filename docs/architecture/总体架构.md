# 总体架构

## 1. 分层视图

```text
Vue 3 B/S 界面
  → FastAPI API / Auth / RBAC / SSE
  → Agent Orchestrator
     → Input Guard
     → Intent & Complexity Router
     → Structured Planner
     → Evidence Manager / RCA
     → Public Reason Composer
  → Deterministic Safety Guardrail（最终否决）
     → Schema / Tool / 参数 / 路径 / 角色 / 模式校验
     → Risk Engine / Approval Manager
  → MCP Client → MCP Tool Registry
  → Restricted Executor
     → dry-run → backup → execute → verify → rollback
  → 麒麟 V11 / Linux 能力层

横切：SQLite WAL、审计哈希链、知识库、配置、脱敏、可观测性
```

## 2. 执行权分离

LLM Provider 位于编排层，输出只能是候选 `ActionPlan` 或注册 Tool 的候选参数。它无权创建 Tool、设置风险、签发审批令牌或调用 subprocess。Guardrail 独立形成内部授权对象，Restricted Executor 只接受该对象，因此 LLM 原始输出不存在直达执行器的路径。

## 3. 部署边界

- 浏览器只访问 API，不连接 MCP 或 OS。
- API 服务账号与执行账号分离。
- MCP Registry 与传输适配分离，协议或 SDK 变化不影响内部 Tool 契约。
- 生产前端使用预构建静态资源；Docker 可选但不是唯一部署方式。
- Secret 位于进程环境或系统 Secret，配置文件只保存引用。

## 4. 推荐工程结构

Phase 1 起按职责建立 `backend/app/api`、`agent`、`guardrails`、`mcp_client`、`executor`、`knowledge`、`audit`、`auth`、`db`、`llm` 和 `core`。Phase 2 建立独立 `mcp_server/tools` 与 `schemas`。当前 Phase 0 不创建空业务目录。

## 5. 技术约束

- Python 3.10+、FastAPI、Pydantic、SQLAlchemy、Alembic、SQLite WAL。
- Vue 3、TypeScript、Vite、Pinia、Vue Router、Element Plus、ECharts。
- 官方 MCP Python SDK；如 LoongArch 不兼容，通过传输适配隔离和能力探测降级。
- 优先 `/proc`、`/sys` 与固定系统命令；统一 `subprocess.run(argv, shell=False)`。
- SQLite FTS5 为知识检索基线，向量检索仅为可选增强。
