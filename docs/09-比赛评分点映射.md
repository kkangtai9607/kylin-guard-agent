# 比赛评分点映射

| 评分方向 | Phase 0 设计 | 后续实现证据 | 演示位置 |
|---|---|---|---|
| OS 感知与 MCP | 总体架构、Tool 元数据 | 能力探测、只读 Tool 测试 | CPU/磁盘/服务证据卡 |
| 自然语言交互 | 状态机、API、结构化计划 | 任务创建与事件流测试 | 输入故障目标、展示计划 |
| 安全护栏 | 安全说明、威胁模型、风险等级 | 注入、RBAC、路径和审批测试 | 直接/间接注入被阻断 |
| 智能根因分析 | 业务闭环、ER | Top 3 与证据评分测试 | 多源证据和置信度 |
| 双层安全审计 | 确定性最终否决、可选审计模型 | 模型失败和意图偏移测试 | 模型建议仍被规则拒绝 |
| 最小权限执行 | ADR、MCP 契约 | 服务账号、固定 argv、sudoers | 安全日志清理 |
| 备份/验证/回滚 | 状态机、执行实体 | 失败注入和回滚集成测试 | 展示验证或预置回滚 |
| 防篡改审计 | ER 哈希链 | 链完整性和篡改检测 | 任务时间线与校验结果 |
| 麒麟与 LoongArch | 兼容矩阵 | Phase 9 真机命令和报告 | 平台能力探测 |
| 文档和演示 | 需求、设计、API、脚本 | 九项交付物一致性检查 | 6 分 30 秒主流程 |

## 需求追踪

- 所有演示状态必须对应真实状态机事件，不展示隐藏思维。
- DEMO 数据和离线 LLM 夹具必须明确标识，不能冒充真实系统或在线模型结果。
- L3 演示仍执行服务端审批令牌校验、备份、验证和审计。
- L4 请求必须在 Tool 调用前阻断。
- 最终提交前重新核对赛事群和官方通知，若评分结构变化需记录计划变更原因。

## 九项交付物

1. 软件功能需求分析。
2. 软件功能设计。
3. 软件产品说明书。
4. 软件功能测试报告。
5. 软件性能测试报告。
6. 安装包和部署文档。
7. 软件源代码。
8. 演示 PPT。
9. 不超过 7 分钟的功能演示视频。
