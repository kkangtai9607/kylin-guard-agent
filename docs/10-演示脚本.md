# 7 分钟演示脚本

## 1. 时间安排

| 时间 | 内容 | 必须展示的证据 |
|---|---|---|
| 0:00–0:30 | 产品定位、平台与 DEMO 标识 | 当前模式、演示数据标签 |
| 0:30–1:15 | 请求读取凭据 | L4、Input Guard 阻断、Tool 调用数为 0 |
| 1:15–3:15 | 磁盘告警诊断 | 多源证据、结构化计划、Top 3 根因 |
| 3:15–5:45 | 安全日志清理 | dry-run、人工审批、备份、执行、验证 |
| 5:45–6:45 | 日志中的间接注入 | `UNTRUSTED_DATA` 标识、恶意文本未改变策略 |
| 6:45–7:00 | 审计总结 | 时间线、哈希完整性、结束页 |

主流程按 6 分 30 秒排练，保留 30 秒机动时间；现场讲解不得超过 7 分钟。

## 2. 稳定性准备

- DEMO 使用固定种子、隔离目录和可一键恢复的场景快照。
- 演示前执行健康检查，验证数据库、API、MCP、演示数据和审计链。
- 远程 LLM 设短超时和有限重试；不可用时切换到 `offline_fixture=true` 的预录结构化响应。
- 离线夹具只替代模型生成，Guardrail、MCP、审批、备份、验证和审计仍真实运行。
- 准备成功验证和验证失败回滚两个预置场景，主演示默认选最稳定的成功路径。
- 页面常驻显示运行模式、DEMO 标识和任务状态，避免口头声明代替证据。

## 3. 安全演示检查点

- LLM 输出没有直接进入执行器。
- 没有任意 Shell Tool。
- L3 未审批时执行按钮和后端接口均拒绝。
- 备份失败不能执行；验证失败进入回滚。
- 不展示隐藏思维，只展示结构化计划和公开理由。
- 日志中的恶意命令仅作为证据文本。

## 4. 演示后恢复

停止演示任务，恢复隔离场景快照，校验演示数据库与审计链，保留脱敏结果用于复盘。不得清理或修改真实系统资源。
