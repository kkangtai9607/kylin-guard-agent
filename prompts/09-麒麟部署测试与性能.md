# Phase 9：麒麟部署、LoongArch 适配和完整测试

```text
本次只执行 Phase 9。完成后停止。

目标：

1. 完整单元、集成、安全和性能测试；
2. 麒麟高级服务器 V11 部署；
3. LoongArch 兼容检查；
4. install.sh；
5. uninstall.sh；
6. healthcheck.sh；
7. systemd unit；
8. Nginx 配置；
9. 最小 sudoers 模板；
10. 依赖安装说明；
11. 前端预构建；
12. 离线或模型不可用降级；
13. 故障场景初始化与重置脚本；
14. 性能测试报告数据。

兼容要求：

- 检测架构和操作系统；
- 不依赖 x86 专属 wheel；
- 可选依赖失败时安全降级；
- Docker 不是唯一部署方式；
- 能检测 dnf/yum/apt；
- 能探测 ss/netstat、lsof、iostat、journalctl、systemctl；
- 安装脚本可重复执行；
- 服务不以 root 长期运行；
- sudoers 最小化；
- 前端运行不依赖 Node；
- requirements 不包含本地绝对路径；
- 提交包不含 site-packages 和 node_modules。

性能报告必须来自真实脚本：

- 首页 API；
- system_snapshot；
- Tool 调用；
- 完整只读诊断；
- 并发任务；
- 审计写入；
- 内存；
- 磁盘；
- 含 LLM 与不含 LLM 延迟。

不得伪造麒麟或 LoongArch 测试。  
如果当前机器不是目标环境，明确区分：

- 已在当前 Linux/Windows/WSL 验证；
- 已通过静态兼容检查；
- 待麒麟 LoongArch 实机验证。

验收：

- 完整测试执行；
- 安全扫描通过；
- 安装和健康检查脚本可运行；
- 部署文档与代码一致；
- 生成真实性能结果；
- 记录所有未验证项。

更新状态后停止。
```
