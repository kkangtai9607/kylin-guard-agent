# Phase 6：受控写操作、验证与回滚

```text
本次只执行 Phase 6。该阶段风险最高，先审查现有安全护栏，再实现受控写 Tool。完成后停止。

只实现以下 Tool：

1. safe_log_cleanup
2. service_restart
3. config_safe_update
4. terminate_process
5. rollback_change

共同要求：

- CONTROLLED_EXECUTION 模式才可用；
- 固定 Tool，不提供通用 Shell；
- 严格参数 Schema；
- L3 强制人工审批；
- dry-run；
- 参数哈希绑定审批令牌；
- 最小权限；
- 执行前备份；
- 执行后验证；
- 失败回滚；
- 幂等性；
- 超时；
- 输出限制；
- 完整审计。

safe_log_cleanup：

- 只接受系统生成的候选 ID，不接受任意路径；
- 区分普通日志、关键数据库日志、审计日志和正在占用文件；
- 默认归档或压缩；
- protected paths 强制拒绝；
- 执行后验证释放空间。

service_restart：

- 服务名必须在白名单；
- 先获取状态和依赖；
- 重启后执行健康检查；
- 失败时记录并给出恢复方案。

config_safe_update：

- 仅白名单配置；
- 备份；
- 展示 diff；
- 语法校验；
- 原子替换；
- 服务健康验证；
- 失败回滚。

terminate_process：

- 禁止 PID 1；
- 禁止 Agent 自身；
- 禁止关键系统进程；
- 先 TERM；
- 等待后再按严格策略决定是否 KILL；
- 验证进程状态。

rollback_change：

- 使用 change_id；
- 校验权限和任务关系；
- 恢复备份；
- 验证；
- 审计。

开发和测试优先使用 DEMO 沙箱，不要在开发机真实破坏系统。

必须提供：

- 无害演示执行器；
- 故障注入；
- 回滚集成测试；
- 审批篡改测试；
- 重复执行测试；
- 执行中断测试。

完成后进行专门安全审查，更新状态和文档，然后停止。
```
