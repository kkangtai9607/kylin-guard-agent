# Phase 5：根因分析与知识库

```text
本次只执行 Phase 5：多源证据根因分析和轻量知识库。完成后停止。

实现：

1. Evidence 标准模型；
2. 指标、进程、日志、端口、服务、配置证据关联；
3. RCA Rule Engine；
4. 可解释置信度评分；
5. Top 3 根因候选；
6. 证据引用；
7. 根因不足提示；
8. SQLite FTS5 知识库；
9. 文档、SOP、工具说明和案例模型；
10. 历史案例检索；
11. 已解决任务生成候选案例；
12. 候选案例人工审核状态；
13. LLM 只负责归纳表达，不得编造证据；
14. 根因分析 API 和测试。

RCA 评分至少考虑：

- 异常幅度；
- 时间相关性；
- 多源证据支持；
- 规则权重；
- 历史案例相似性；
- 反证和冲突证据。

要求：

- 每个根因必须关联 evidence_id；
- 置信度来自代码计算；
- LLM 不得自己输出未经校准的置信度；
- 无足够证据时不得武断下结论；
- 未审核案例不能直接触发高风险操作；
- 知识库内容标记为 UNTRUSTED_DATA；
- 知识库不得覆盖安全规则。

演示最少支持：

- 磁盘空间异常；
- 高 CPU；
- 僵尸进程；
- 服务启动失败；
- 配置漂移。

验收：

- 每个根因均可追踪证据；
- FTS5 搜索可用；
- 候选案例可审核；
- 安全边界未被知识库绕过；
- 测试通过。

更新文档和状态后停止。
```
