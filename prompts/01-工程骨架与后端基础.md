# Phase 1：工程骨架与后端基础

```text
阅读 @AGENTS.md、@PROJECT_SPEC.md、@PLANS.md、@CURRENT_STATUS.md 和 Phase 0 架构文档。

本次只执行 Phase 1：工程骨架和后端基础。完成后停止，不得进入 Phase 2。

目标：

1. 创建 backend、mcp_server、frontend、config、demo、deploy、scripts、docs 的工程骨架；
2. 创建 FastAPI 应用；
3. 实现配置加载；
4. 实现统一响应和错误码；
5. 实现 SQLite WAL；
6. 实现 SQLAlchemy 模型和 Alembic 初始迁移；
7. 实现用户、角色、会话和基础 RBAC；
8. 实现登录接口；
9. 实现 Agent Task 和 Task Step 基础模型；
10. 实现任务状态机；
11. 实现审计事件基础写入和查询；
12. 实现健康检查；
13. 建立 pytest、ruff、mypy 配置；
14. 建立安全扫描脚本；
15. 提供最小前端骨架，但不实现完整页面。

安全约束：

- 不得出现 shell=True、eval、exec；
- 不得提供任意命令执行接口；
- 密钥只能从环境变量读取；
- 密码必须使用可靠哈希；
- 默认运行模式不得开放写操作；
- API 错误不得泄露敏感信息；
- 审计 payload 必须有脱敏入口。

验收：

- 后端能启动；
- 数据库迁移能执行；
- 能创建初始管理员；
- 能登录并访问受保护接口；
- 能创建任务并合法转换状态；
- 非法状态转换被拒绝；
- 审计事件可查询；
- pytest 通过；
- ruff 通过；
- mypy 在当前配置范围内通过；
- security_scan 通过。

请先检查已有实现，再给出简短实施计划，然后编码和测试。  
完成后更新 CURRENT_STATUS.md 和 PLANS.md，并停止。
```
