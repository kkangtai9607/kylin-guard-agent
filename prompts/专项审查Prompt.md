# Codex 专项审查 Prompt

## 1. 全仓库架构审查

```text
进入 Plan/Review 模式。

阅读 AGENTS.md、PROJECT_SPEC.md、CURRENT_STATUS.md 和整个仓库。

只做审查，不修改代码。检查：

1. 项目是否真正形成“感知→计划→校验→审批→执行→验证→回滚→审计”闭环；
2. 模块职责是否清晰；
3. 是否存在重复实现；
4. 状态机是否完整；
5. API、数据库、前端和文档是否一致；
6. MCP 是否真正用于工具调用；
7. DEMO 与真实数据是否混淆；
8. 麒麟和 LoongArch 是否存在明显硬依赖问题；
9. 比赛评分点是否有缺口；
10. 7 分钟演示是否稳定。

按严重程度输出：

- Blocker
- High
- Medium
- Low

每个问题给出文件路径、证据、影响和最小修复建议。不要直接修改。
```

## 2. 安全红队审查

```text
只做防御性安全审查，不修改代码。

重点检查：

- 任意 Shell；
- shell=True、eval、exec；
- LLM 输出直达执行器；
- Tool 参数注入；
- 路径穿越；
- 符号链接逃逸；
- protected paths；
- sudoers 过宽；
- 审批令牌重放、篡改和过期；
- 用户目标与动作偏移；
- 只读任务升级为写操作；
- 直接和间接提示词注入；
- 日志、配置和知识库信任边界；
- 凭据泄露；
- 审计绕过；
- 备份、验证和回滚缺失；
- Agent 终止自身或关键进程；
- 模型故障时 fail-open。

输出威胁场景、代码证据、风险等级和修复建议。  
不要提供攻击生产系统的操作步骤。
```

## 3. 修复审查问题

```text
阅读上一轮审查报告。

只修复我明确选中的问题编号。  
先给出最小改动计划，再修改、测试和更新文档。  
不得借机大范围重构。  
完成后报告修复证据，并停止。
```

## 4. 文档一致性审查

```text
对照实际代码检查所有 docs 文档。

标记：

- 已实现且描述准确；
- 已实现但描述过时；
- 文档声称实现但代码不存在；
- 代码已实现但文档缺失；
- 测试报告无可复现命令；
- 性能数据缺少来源；
- 部署步骤无法从仓库复现。

只输出差异报告，不修改文件。
```

## 5. 发布前最终检查

```text
执行发布前只读检查：

1. Git 状态；
2. 密钥和凭据扫描；
3. shell=True/eval/exec 扫描；
4. 通用 Shell Tool 扫描；
5. 后端测试；
6. 前端类型检查和构建；
7. 安全测试；
8. 安装包内容；
9. node_modules/site-packages/cache/log 排除；
10. 文档与代码一致性；
11. 默认运行模式；
12. DEMO 重置；
13. 九项提交材料；
14. 已知限制。

不要修改代码。输出 Go/No-Go 结论及阻断项。
```
